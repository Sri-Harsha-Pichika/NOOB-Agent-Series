# Contributing to NOOB Agents

We welcome contributions from the community! Whether it's bug fixes, new features, documentation improvements, or workflow diagrams, your help makes NOOB Agents stronger.

---

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
   - Branch naming convention:
     - `feature/<short-description>` for new features.
     - `fix/<short-description>` for bug fixes.
     - `docs/<short-description>` for documentation updates.

2. **Commit messages** should follow [Conventional Commits](https://www.conventionalcommits.org/):
   - `feat: add HYBRID developer mode docs`
   - `fix: correct NOOB++ workflow diagram`
   - `docs: update README with examples`

3. **Pull Requests (PRs):**
   - Keep PRs small and focused.
   - Include a clear description of changes.
   - Link related issues if applicable.
   - Update documentation and examples when relevant.

---

## Contribution Checklist
- [ ] Code or docs build/run correctly.
- [ ] Tests/examples updated if applicable.
- [ ] Documentation updated (README, KBA, diagrams).
- [ ] Commit messages follow conventions.
- [ ] PR description is clear and complete.
